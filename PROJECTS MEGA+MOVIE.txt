createAccount -> for create account

createEmailPasswordSession: {only use for login}

Creates a session for a user using their email and password.
Essentially, it logs the user in.
Returns session data:
userId → ID of the logged-in user
sessionId → Unique ID of this session
secret / token info → Used to authorize API calls
How it works: ->
User enters email & password.
Appwrite checks if credentials are valid.
If valid, it creates a session in the server.
Your frontend receives session info and can now make authenticated requests.

deleteSession:

deleteSession(sessionId)
Deletes one specific session.
Example: Logging out a user on one device but leaving other devices logged in.
You need to provide the sessionId (from createEmailSession).
deleteSessions()
Deletes all sessions of the user.
Example: Logging out from every device, like a “Log out of all devices” feature.
Why use them:
Removing sessions revokes access tokens, so the user can’t make API requests anymore.
Keeps your app secure by controlling who is logged in.
Analogy:
deleteSession → “Take away the key from one device.”
deleteSessions → “Take away all keys from every device.”


Session: like time period that user should login after that logout..
Users don’t have to send their password every time.
Once logged in, users stay logged in for a while.
Appwrite uses the session to identify the current user.
When the session is deleted, the server immediately knows the user is logged out.
