I want to build a movie website using Vite + React + JavaScript where I strictly use props, custom hooks, Redux Toolkit for global state management, lazy loading, useMemo, useCallback, useState, useEffect, React Hook Form (useForm), localStorage, protected routing, dependency injection where needed, and API data fetching, and I do not want to use Context API at all. When the app loads, the Home page should appear first with a header containing a Home button and a Sign In button; after login, the Sign In button should automatically change to Logout. The Home page should fetch movies from an API and display them in two horizontal rows, one moving left-to-right and the other right-to-left. The Sign In and Sign Up forms must be built using useForm with proper validation and submission handling. After successful authentication, the user should be redirected to a protected page. On this protected page, display “Hello username” and an input field with a button where the user can enter movie names separated by commas or dots (e.g., dhoom, om shanti om, koi mil gya). The input should be parsed, each movie searched via API, and results displayed as movie cards under tags; after every three movies, the list should be horizontally scrollable/movable. The protected route must only be accessible to logged-in users, and if a user is already logged in, they should automatically navigate to this page on reload. Logout should clear authentication and redirect to Home. Each user’s movie searches must be saved in localStorage per user, so when the same user logs in again, their previous data is restored, while other users must not see it. Use Redux Toolkit for authentication and movie state, useMemo for expensive computations, useCallback for stable handlers, lazy loading for pages, and apply dependency injection where suitable, keeping the architecture clean, component-based, and driven only by the specified tools.
